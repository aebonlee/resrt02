# 2026-06-14 · 배포 경로 수정 (커스텀 도메인 자산 404 해결)

## 증상
- 커스텀 도메인 `rest02.dreamitbiz.com`은 200이지만 화면이 비어 있음.
- `index.html`이 JS/CSS를 `/resrt02/assets/...` 로 참조 → **404**.

## 원인
- `vite.config.js`의 `base: '/resrt02/'` (게다가 리포명은 `rest02`, base는 오타 `resrt02`).
- 커스텀 도메인은 **루트(`/`)로 서빙**되므로 base가 `/resrt02/`면 자산 경로가 어긋남.

## 조치
- `vite base` → **`/`** (HashRouter 사용이라 deep-link는 해시로 처리, 404.html 불필요).
- `index.html` favicon `/resrt02/favicon.svg` → `/favicon.svg`.
- `public/CNAME`(`rest02.dreamitbiz.com`) 추가 — `npm run deploy`(gh-pages) 시 커스텀 도메인 유지.
- `package.json` name `resrt02`→`rest02`, homepage → `https://rest02.dreamitbiz.com/`.
- README 오타·배포 안내 정리.

## 검증
- `npm run build` 후 `dist/index.html` 자산이 `/assets/...`(루트)로 참조됨, `dist/CNAME` 포함.
- `npm run lint` 통과.
